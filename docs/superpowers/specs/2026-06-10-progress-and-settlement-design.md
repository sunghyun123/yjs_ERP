# 공사이력 페이지 + 기성/준공 탭 설계

> 작성일: 2026-06-10

---

## 1. 배경 및 목표

기존 수주 수정 Dialog에 기성·준공 체크박스만 있고 금액 입력란이 없었다. 기성은 공사당 여러 건이 발생하고 차수마다 금액이 다르며, 공사이력은 투입실적 다음으로 자주 쓰는 기능이다. 이를 개선한다.

**핵심 원칙:** 자주 쓰는 기능은 직접 접근, 가끔 쓰는 기능은 Dialog 탭

---

## 2. 변경 범위

### 신규 페이지
- `/progress` — 공사이력 전용 페이지 (사이드바 메뉴 추가)

### 기존 변경
- 수주 수정 Dialog: 탭 구조 도입 `[기본정보] [기성] [준공]`
- 기본정보 우측 패널: 준공 관련 필드 제거 (준공 탭으로 이동)

## 3. 공사이력 페이지 (`/progress`)

### 3-1. 입력 탭 (기본 탭)

**레이아웃:** 투입실적 페이지(`/input`)와 동일한 2단 패턴 — 좌측 입력 폼 + 우측 sticky 계산 패널

**흐름:**
1. 지중No 또는 공사명으로 공사 검색 (SearchableSelect)
2. 공사 선택 즉시 → 해당 공사의 **가장 최근 공사이력 1건** 자동 로드하여 표시
   - 표시 항목: 마지막 작업일자, 누계 성과금액, 현재 달성율(%)
   - 목적: 중복 등록 방지, 이어서 입력할 기준점 제공
3. 입력 필드: 작업일자(date), 성과금액(증분, MoneyInput)
4. 저장

**우측 계산 패널 (실시간):**
- Δ달성율(%) = 성과금액 / 하도적용금액 × 100
- 저장 후 누계 성과금액 = 기존 누계 + 입력값
- 저장 후 달성율 = 누계 성과금액 / 하도적용금액 × 100
- 참고값: 하도적용금액, 수주금액

**계산 참고:**
```
하도적용금액 = 수주금액_공급가 × (1 - 보험료율) × 하도전용율
Δ달성율(%)  = 성과금액(증분) / 하도적용금액 × 100
저장 후 달성율 = 기존누계달성율 + Δ달성율
```

**DB 저장:** `공사이력` 테이블에 INSERT
- `수주_id`, `작업일자`, `성과금액`(증분 금액 그대로)
- 달성율은 DB에 저장하지 않음 (화면 표시용 역산)

### 3-2. 현황 탭

**레이아웃:** DataTable (투입실적 현황 탭과 동일 패턴)

**필터:** 공사 선택 드롭다운, 날짜 범위

**테이블 컬럼:** 공사명, 작업일자, 성과금액, 누계 달성율, 수정/삭제

**수정:** 행 클릭 또는 수정 버튼 → Sheet에서 수정 (작업일자, 성과금액)

## 4. 수주 수정 Dialog 탭 구조

기존 단일 폼 → 3탭으로 분리

```
[기본정보] [기성 N] [준공]
```

- 탭 배지: 기성 탭에 등록 건수 표시 (예: `기성 3`)
- 준공 탭: 완료 여부에 따라 배지 표시 (`완료` / 미표시)

### 4-1. 기본정보 탭

기존 OrderForm 우측 패널에서 **준공 관련 필드 제거** (준공여부, 준공일, 준공액_공급가).
나머지 폼 구조 그대로 유지.

### 4-2. 기성 탭

**테이블:**
| 차수 | 기성일 | 공급가 | 부가세(10%) | 합계 | 수정/삭제 |
|------|--------|--------|------------|------|----------|
| 1차 | ... | ... | ... | ... | |
| 누계 | | ... | ... | ... | |

**추가 폼 (인라인, `+ 기성 추가` 클릭 시 펼쳐짐):**
- 차수: 자동 (현재 최대 차수 + 1)
- 기성일: date input
- 기성액_공급가: MoneyInput → 부가세(×0.1), 합계 실시간 표시
- 저장 / 취소

**수정:** 각 행 "수정" 클릭 → 동일 인라인 폼 (해당 행 데이터 채워짐)

**DB:** `기성` 테이블 (수주_id, 차수, 기성일, 기성액_공급가)

### 4-3. 준공 탭

**필드:**
- 준공완료 체크박스
- 준공일 (date, 준공완료 체크 시 활성)
- 준공액_공급가 (MoneyInput, 준공완료 체크 시 활성)
- 실시간 표시: 부가세(공급가 × 0.1), 합계(공급가 + 부가세)

**저장:** 준공 탭 자체 저장 버튼 — `수주` 테이블의 준공 컬럼(`준공여부`, `준공일`, `준공액_공급가`)만 UPDATE. 기본정보 탭 저장과 독립적으로 동작.

**체크박스 해제 시:** 준공일, 준공액_공급가 null 처리

**DB:** `수주` 테이블의 `준공여부`, `준공일`, `준공액_공급가` 컬럼 (기존 그대로)

## 6. 구현 순서 (우선순위)

1. **수주 Dialog 탭 구조** — 기존 OrderForm 리팩터링 (기본정보 + 기성 탭 + 준공 탭)
2. **공사이력 페이지** — `/progress` 신규 페이지 (입력 탭 + 현황 탭)
3. **사이드바** — 공사이력 메뉴 추가
