# 투입실적 입력 화면 UI/UX 개선 설계

**날짜:** 2026-06-10  
**대상 파일:** `src/app/(dashboard)/input/_components/InputForm.tsx`

## 배경

기획팀 요청: 테이블 레이아웃 과도한 여백, 입력 필드 시인성 부족, 행 가독성 개선.

## 확정된 설계

### 1. 레이아웃 압축 (max-w-2xl + 중앙 정렬)

- 폼 최상위 flex 컨테이너(`flex flex-col lg:flex-row`)에 `max-w-2xl mx-auto` 추가
- 테이블이 화면 전체 너비를 채우지 않고 중앙에 집중 배치
- 좌측(입력) + 우측(계산 패널) 두 컬럼이 함께 압축

### 2. 입력 필드 시인성 강화

| 속성 | 현재 | 변경 후 |
|------|------|---------|
| 크기 | `w-20 h-9` | `w-[68px] h-10` |
| 기본 테두리 | `border-gray-200` | `border-gray-400` |
| 포커스 링 | `ring-[#3d5af1]/20` | `ring-[#3d5af1]/40` |

### 3. 행 가독성 — 지브라 + 호버 둘 다

- 짝수 행: `even:bg-blue-50/40` (항상 보이는 줄 구분)
- 호버 시: `hover:bg-blue-100/50` (지브라 위에 덮어씀)
- `divide-y divide-gray-50` → `divide-y divide-gray-100` (구분선 약간 강화)

### 기존 유지 항목

- 값이 있는 인풋의 파란/인디고 배경 강조 (`주값 > 0`, `야값 > 0`)
- 모든 값이 0인 행의 `opacity-35` 처리
- 우측 계산 패널, 저장/초기화 버튼 구조
- 공사 선택 자동완성, 투입일 입력 영역
