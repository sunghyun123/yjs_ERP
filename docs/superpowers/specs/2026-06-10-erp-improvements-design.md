### 1-2. ERP 미입력 공사 섹션

기존 "진행중 공사 현황" 섹션을 제거하고 "ERP 미입력 공사" 섹션으로 교체.

**동작 흐름:**
1. 외부 대시보드(www.yjsboard.com)가 `POST /api/dashboard-sync`로 전날 등록된 공사 목록을 전송
2. ERP는 수신된 공사를 `dashboard_공사` 테이블에 누적 저장 (중복 무시)
3. 홈 화면은 `dashboard_공사` 중 `삭제됨 = false`인 항목을 표시
4. 각 항목마다 `dashboard_공사.지중no` → `수주.지중no`로 `수주_id`를 조회한 뒤, 해당 `진행날짜`에 `공사이력.작업일자` AND `투입실적.투입일` 둘 다 등록된 경우 자동 제거. 수주 미등록 공사(지중no 매칭 실패)는 미입력 상태로 유지.

**상태 표시 방식 (요약 뱃지 + Outline 버튼):**

| 상태 | 뱃지 | 표시 버튼 |
|---|---|---|
| 둘 다 미입력 | 🔴 `✗ 이력·실적 미입력` | 이력 입력 / 실적 입력 / ✕ |
| 공사이력만 있음 | 🟡 `⚠ 실적 미입력` | 실적 입력 / ✕ |
| 투입실적만 있음 | 🟡 `⚠ 이력 미입력` | 이력 입력 / ✕ |
| 둘 다 있음 | _(목록에서 자동 제거)_ | — |

- ✕ 버튼: `삭제됨 = true`로 소프트 삭제 (취소 공사 처리용)
- "투입실적 입력", "공사이력 입력" 버튼: 해당 수주의 입력 페이지로 이동
- 페이지당 10건, 하단 페이지 이동

**"이력 입력" / "실적 입력" 버튼 이동 경로:**
- 이력 입력 → `/progress?수주_id=...&날짜=...` (공사이력 입력 폼 사전 채움)
- 실적 입력 → `/input?수주_id=...&날짜=...` (투입실적 입력 폼 사전 채움)

### 1-3. API 엔드포인트

```
POST /api/dashboard-sync
Authorization: Bearer {DASHBOARD_API_KEY}
Content-Type: application/json

{
  "constructions": [
    {
      "지중no": "TY25-007",
      "공사명": "성동 케이블 교체",
      "진행날짜": "2026-06-09"
    }
  ]
}
```

- `DASHBOARD_API_KEY`는 환경변수로 관리
- 동일 `지중no` + `진행날짜` 조합이 이미 존재하면 무시 (멱등성 보장)
- 응답: `{ "inserted": N, "skipped": M }`

## 2. 공무 전용 화면

### 2-1. 페이지 구조

```
/gongmu              → 공무 선택 페이지 (공무담당자 카드 목록 + 종합 현황 카드)

/gongmu/total        → 종합 현황
```

인증과 무관하게 누구나 접근 가능 (선택형). 사이드바에 "공무 보고서" 메뉴 추가.

---

### 2-2. 개인 공무 화면 (`/gongmu/[id]`)

**월간 요약 (상단 카드):**
- 월간계획 (공사) / 누적실적 / 달성률 — 프로그레스 바 포함
- 월간계획 (공무) / 누적실적 / 달성률 — 프로그레스 바 포함
- 이번주 금주실적 합산

**주차 탭:**  
해당 월의 주차 목록 (1주차~5주차). 현재 주차 기본 선택.  
주차 계산: ISO 주차 기준. `week_no`와 `year`는 모두 ISO year 기준으로 저장 (12월 말 주차가 다음 해 ISO year로 넘어가는 경우 포함). 월 경계 처리 유틸 함수로 추상화.

**공사 파트 테이블:**

| 지중No | 금주 작업 | 차주 작업 | 금주계획 | 금주실적 | 차주계획 | 비고 |
|---|---|---|---|---|---|---|
| (수주 연동 or 직접 입력) | 텍스트 | 텍스트 | 금액 | 금액 | 금액 | 텍스트 |

**공무 파트 테이블:**

| 번호 | 금주 작업 | 차주 작업 | 금주계획 | 금주실적 | 차주계획 | 비고 |
|---|---|---|---|---|---|---|
| 순번 | 텍스트 | 텍스트 | 금액 | 금액 | 금액 | 텍스트 |

- 행 추가/삭제 가능
- 공사 파트 지중No: 수주대장 검색 후 선택(공사명 자동완성) 또는 직접 입력
- 모든 셀 자유 편집 가능 (ERP 초안 덮어쓰기 포함)

**ERP 자동 초안 채움:**  
해당 주차에 `담당공무_id = 현재 공무`인 공사이력/기성이 있으면, 주간보고 행을 초안으로 자동 생성:
- 금주실적 ← 공사이력.성과금액 또는 기성.기성액_공급가
- 금주작업 ← 공사이력.작업내용 또는 기성.작업내용 (입력된 경우)
- 초안 행은 시각적으로 구분 표시 (연한 배경), 저장 전까지 편집 가능
- 저장 후에는 일반 행과 동일하게 취급

**자동 계산 (DB 저장 안 함, 조회 시 계산):**
- 누적실적 = 해당 월 1주차 ~ 현재 주차 이전까지의 금주실적 합산 (전주실적 복붙 불필요)
- 달성률 = 누적실적 ÷ 월간계획금액 × 100

**엑셀 내보내기:**
- 이번주 보고서 (단일 공무, 단일 주차)
- 월간 보고서 (단일 공무, 해당 월 전체 주차)
- 새 포맷 (기존 양식 미재현). 공사/공무 파트 구분, 주차별 금주계획/실적/달성률 표 형식.

### 2-3. 종합 현황 (`/gongmu/total`)

- 공무별 달성률 수평 바 차트
- 공무별 주간보고 테이블 (공사/공무 파트 합산)
- 종합 월간 보고서 엑셀 내보내기

### 2-5. 기존 테이블 수정

#### `공사이력` — 컬럼 추가

| 추가 컬럼 | 타입 | 설명 |
|---|---|---|
| 작업내용 | text \| null | 선택 입력 |
| 담당공무_id | int \| null (FK → 공무담당자) | 기본값: 수주의 공사담당자, 변경 가능 |

#### `기성` — 컬럼 추가

| 추가 컬럼 | 타입 | 설명 |
|---|---|---|
| 작업내용 | text \| null | 선택 입력 |
| 담당공무_id | int \| null (FK → 공무담당자) | 기본값: 수주의 공사담당자, 변경 가능 |

#### `수주` — 컬럼 추가

| 추가 컬럼 | 타입 | 설명 |
|---|---|---|
| 공무담당자_id | int \| null (FK → 공무담당자) | 기존 `공사담당자` 텍스트 필드와 병행 유지 |

기존 `공사담당자` 텍스트 필드는 제거하지 않고 유지. 신규 `공무담당자_id`를 추가하여 담당공무 드롭다운에 기본값으로 사용.
